图文详解 Claude Code 从入门到精通（一）：Claude Code 的安装与启动

2026年3月11日 16:47 阅读约 6 分钟实用工具教程 0 次浏览 0 个赞 0 条评论
工具 技术
摘要
Claude Code是Anthropic推出的AI编程命令行工具，专为开发者提供终端内AI协作工作流。本文作为系列入门指南的第一篇，重点介绍其安装与启动流程。文章详细说明了系统要求，包括操作系统、Node.js版本和网络条件，并提供了原生安装和包管理器安装两种方法。针对国内用户，推荐使用智谱GLM Coding Plan以获得更稳定的网络体验和更高性价比，同时给出了具体的配置步骤和模型切换方法。

前言

在 AI 辅助编程工具飞速发展的今天，Claude Code 作为 Anthropic 推出的命令行编程助手，正在被越来越多的开发者所青睐。与传统的桌面应用不同，Claude Code 直接运行在你的终端中，能够深入理解你的代码库，帮助你完成编码、调试、重构等任务。

本系列文章将带你从零开始掌握 Claude Code，第一篇我们专注于安装与启动。

一、什么是 Claude Code？

Claude Code 是 Anthropic 推出的 AI 编程命令行工具（CLI）。它不同于网页版的 Claude 或桌面应用，Claude Code 设计初衷是为开发者提供一个可以在终端中直接与 AI 协作的工作流。

Claude Code 的核心能力包括：

  • 代码理解与分析：深入分析你的代码库，提供上下文相关的建议
  • 自动化编码：直接帮你创建、修改代码文件
  • Git 集成：对话式完成 Git 操作
  • MCP 扩展：支持外部服务集成，扩展能力
  • 终端工具：直接在终端中运行命令、运行测试

二、系统要求

在安装之前，先确认你的环境满足以下要求：

要求规格
操作系统macOS 13.0+ / Windows 10 1809+ / Ubuntu 20.04+ / Debian 10+
Node.js安装 Node.js 18 或更新版本环境
网络需要稳定互联网连接
ShellBash 或 Zsh（效果最佳）

Windows 用户注意：需要先安装 Git for Windows。

三、安装步骤

Claude Code 提供多种安装方式，推荐使用官方原生安装器。

方法一：原生安装（推荐）

macOS / Linux / WSL：

curl -fsSL https://claude.ai/install.sh | bash

Windows CMD：

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

方法二：包管理器安装

macOS（Homebrew）：

brew install --cask claude-code

Windows（npm）：

npm install -g @anthropic-ai/claude-code

验证安装

安装完成后，验证是否成功：

claude --version

如果输出版本号，说明安装成功。

四、配置 Claude Code

如果你使用的是官方 Claude 订阅，首次使用 claude 命令时会引导你在浏览器中完成登录。登录凭证会被缓存，后续无需重复登录。如需切换账户，可使用 /login 命令。

如果你在国内使用，推荐接入智谱 GLM Coding Plan套餐，可以获得更稳定的网络体验和更高的性价比。

为什么选择 GLM Coding Plan？

  • 价格实惠：相比官方 Claude API，GLM Coding Plan 价格更低，Pro 版每月仅需约 100 元
  • 网络稳定：国内直连，无需代理
  • 3 倍用量：搭配 GLM Coding Plan 可获得 3 倍用量
  • 模型对标：
    • GLM-4.5-Air 对标 Claude Haiku
    • GLM-4.7 对标 Claude Sonnet/Opus
    • GLM-5 对标 Claude Opus（复杂任务推荐）

配置步骤（以智谱 GLM 为例）

步骤 1：注册并获取 API Key

  1. 访问 智谱开放平台，注册账号并登录

  2. 在导航栏找到「API Keys」，创建一个新的 API Key

  3. 务必妥善保存你的 API Key

注意：首次使用需要先订阅 GLM Coding Plan 套餐

步骤 2：配置环境变量

配置 Claude Code 使用智谱 GLM 模型，有以下三种方式：

方式一：Coding Tool Helper（推荐）

这是智谱官方提供的自动化助手，一条命令完成所有配置：

npx @z_ai/coding-helper

运行后按照界面提示操作，即可自动完成安装、套餐配置和 MCP 服务器管理。

方式二：自动化脚本（仅 MacOS/Linux）

curl -O "https://cdn.bigmodel.cn/install/claude_code_env.sh" && bash ./claude_code_env.sh

脚本会自动配置 ~/.claude/settings.json 文件。

方式三：手动配置（支持 MacOS/Linux/Windows）

对于 MacOS/Linux：

# 编辑配置文件
nano ~/.claude/settings.json

添加或修改以下内容（将 your_zhipu_api_key 替换为你的 API Key）：

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your_zhipu_api_key",
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "API_TIMEOUT_MS": "3000000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": 1
  }
}

同时编辑 ~/.claude.json 文件：

nano ~/.claude.json

添加：

{
  "hasCompletedOnboarding": true
}

对于 Windows：

# 编辑配置文件
notepad %USERPROFILE%\.claude\settings.json

添加同样的内容（注意路径为 %USERPROFILE%\.claude\settings.json）。%USERPROFILE% 是你的用户目录，一般在C盘下。

步骤 3：验证配置

重新打开一个终端窗口，运行：

claude --version

如果显示版本号，说明安装成功。启动 Claude Code：

claude

首次启动时，若提示「Do you want to use this API key」，输入 Yes 确认。

切换不同模型

配置完成后，默认的模型映射关系如下：

Claude 模型GLM 模型
OpusGLM-4.7
SonnetGLM-4.7
HaikuGLM-4.5-Air

如需使用 GLM-5（对标 Opus，适合复杂任务），编辑 ~/.claude/settings.json：

{
  "env": {
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-4.7",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5"
  }
}

注：GLM-5 在高峰期（每日 14:00-18:00）按 3 倍用量计算，非高峰期按 2 倍计算。建议复杂任务使用 GLM-5，普通任务使用 GLM-4.7 以节省用量。

配置不生效？试试这样排查

  1. 关闭所有 Claude Code 窗口，重新打开新的终端再运行 claude
  2. 检查 ~/.claude/settings.json 的 JSON 格式是否正确（可使用在线 JSON 校验工具）
  3. 删除配置文件后重新配置

五、首次启动与基本操作

启动 Claude Code

进入任意项目目录，启动 Claude Code：

cd your-project
claude

你将看到欢迎屏幕，显示会话信息、最近对话和更新提示。

基本命令一览

命令功能
claude启动交互模式
claude "task"运行一次性任务
claude -p "query"运行查询后立即退出
claude -c继续当前目录的最近对话
claude -r恢复之前的对话
/help显示可用命令
/clear清除对话历史
exit 或 Ctrl+C退出

首次对话示例

启动后，你可以尝试以下问题：

帮我分析这个项目的结构

Claude Code 会扫描你的代码库并提供摘要。你也可以问：

我如何在 Claude Code 中创建自定义 skills？

六、配置与自定义

自动更新设置

Claude Code 默认自动更新。如需配置更新渠道，使用 /config 命令或编辑 ~/.claude/settings.json：

{
  "autoUpdatesChannel": "stable"
}
  • latest：默认获取最新功能
  • stable：约一周前的稳定版本

禁用自动更新

如需手动控制更新，可在 shell 配置中添加：

export DISABLE_AUTOUPDATER=1

Windows 特殊配置

在 Windows 上使用 Git Bash 本地运行，需指定 bash 路径：

$env:CLAUDE_CODE_GIT_BASH_PATH="C:\Program Files\Git\bin\bash.exe"

七、常见问题

Q：Claude Code 是否免费？
A：Claude Code 需要付费订阅（Pro、Max、Teams 或 Enterprise）。新用户通常有免费试用额度。

Q：安装时遇到网络问题怎么办？
A：确保网络可以访问 Anthropic 服务。对于企业网络，可能需要配置代理。

Q：如何卸载 Claude Code？
A：本地安装执行 rm -f ~/.local/bin/claude && rm -rf ~/.local/share/claude；Windows 使用控制面板或 WinGet 卸载。

Q：支持中文对话吗？
A：支持，Claude Code 可以用中文进行对话。
